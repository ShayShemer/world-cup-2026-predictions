🏆 World Cup 2026 Predictions App

מגיש: שי שמר קורס: הנדסת פתרונות תוכנה מתקדמים חלק א תאריך: יולי 2026

תיאור הפרויקט

אפליקציית ניחושים למונדיאל 2026 המאפשרת למשתמשים לנחש תוצאות משחקים ולהשוות את ניחושיהם להמלצות מנוע AI סטטיסטי. המערכת מושכת נתונים חיים מ-ESPN API כל 60 שניות ומתעדכנת אוטומטית לאורך כל הטורניר — החל מ-Round of 32 ועד לגמר. ניחושי AI מוצגים מול התוצאה בפועל, צבועים ירוק (צדק) או אדום (טעה).

קבלת הקבצים

מקבלים מהמגיש קובץ World Cup Project.zip ומחלצים אותו לתיקייה לבחירתכם (לדוגמה G:\World Cup Project). את נתיב התיקייה יש לשים ב-Command Prompt בפקודה שבשלב 2 למטה.

הרצת הפרויקט

1
פותחים Docker Desktop ומחכים לסימן הירוק בתחתית
2
פותחים Command Prompt ומריצים:

cd /d "G:\World Cup Project"
docker compose up --build
3
מחכים לשורה:
✅ ALL SYSTEMS READY — Open http://localhost:8080
4
פותחים דפדפן בכתובת http://localhost:8080

ארכיטקטורה — 4 מיקרוסרוויסים

שירות טכנולוגיה פורט תפקיד
stats-service Python / FastAPI 8001 מנוע ניחוש Poisson Distribution עם 5 גורמים משוקללים
api-service Node.js / Express 3001 ניהול משחקים, sync חי מ-ESPN, תוצאות
frontend Nginx + HTML/JS/CSS 8080 ממשק משתמש, reverse proxy ל-API
db PostgreSQL 16 פנימי שמירת ניחושים ותוצאות, נתונים קבועים

כל השירותים רצים על רשת Docker פנימית (app-network). שמות השירותים משמשים כ-hostnames לתקשורת פנימית.

תכונות עיקריות

🤖 מנוע AI — Poisson Distribution

5 גורמים: היסטוריה (20%), פורמה (30%), עוצמה (25%), דירוג FIFA (15%), שחקנים חסרים (10%). כולל ניחוש הארכה ופנדלים.

📡 עדכון חי אוטומטי

ESPN API נסרק כל 60 שניות. תוצאות ושעון המשחק החי (דקה + תוצאה) ומשחקי שלבים מתקדמים נוספים אוטומטית.

✅ AI מול מציאות

בכל משחק שהסתיים מוצג ניחוש ה-AI מול התוצאה האמיתית, צבוע ירוק אם צדק ואדום אם טעה.

📊 גרף הסתברויות

פאנל AI מציג את 8 התוצאות הסבירות ביותר עם אחוזי הסתברות.

🎯 פאנל תוצאות

תפריט צד שמציג את כל המשחקים לפי סיבוב, עם ניחוש AI מול תוצאה בפועל.

🕐 שעון ישראל

כל שעות המשחקים מוצגות בשעון ישראל (IL). במשחק חי מוצג: דקה + תוצאה עדכנית.

נקודות הנדסיות

🔗
depends_on + healthcheck — סדר עלייה מובטח: DB מוכן לפני api-service, stats-service מוכן לפני שליחת בקשות ניחוש.
💾
pgdata volume — נתוני המשתמשים שורדים docker compose down ולא נמחקים בין הפעלות.
🔄
ON CONFLICT DO UPDATE — upsert בטוח בלי כפילויות, גם כאשר ESPN שולח אותם נתונים שוב ושוב.
🔇
healthcheck log filter — בקשות /health מסוננות מהלוגים, הטרמינל נשאר קריא להצגה.
